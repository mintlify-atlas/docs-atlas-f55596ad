## Overview

Migrations are version control for your database, allowing you to define and share the application's database schema. All migration files are located in `database/migrations/`.

## Running Migrations

### Run All Pending Migrations

```bash
php artisan migrate
```

### Rollback the Last Migration

```bash
php artisan migrate:rollback
```

### Rollback All Migrations

```bash
php artisan migrate:reset
```

### Rollback and Re-run All Migrations

```bash
php artisan migrate:refresh
```

### Drop All Tables and Re-run Migrations

```bash
php artisan migrate:fresh
```

## Migration Files

Migrations run in alphabetical/numerical order based on their filename prefixes.

### 1. Create Cache Table

**File:** `0001_01_01_000001_create_cache_table.php`

Creates Laravel's cache storage tables.

```php
public function up(): void
{
    Schema::create('cache', function (Blueprint $table) {
        $table->string('key')->primary();
        $table->mediumText('value');
        $table->integer('expiration')->index();
    });

    Schema::create('cache_locks', function (Blueprint $table) {
        $table->string('key')->primary();
        $table->string('owner');
        $table->integer('expiration')->index();
    });
}
```

**Tables Created:**
- `cache` - Stores cached data
- `cache_locks` - Manages cache locks for atomic operations

```php
public function up(): void
{

```

**Tables Created:**
- `jobs` - Queued jobs waiting to be processed

### 4. Create Usuarios Table

**File:** `002_create_usuarios_table.php`

Creates the usuarios (users) table with role-based access.

```php
public function up(): void
{
    Schema::create('usuarios', function (Blueprint $table) {
        $table->id();
        $table->string('nombre');
        $table->string('password');
        $table->string('email')->unique();
        $table->enum('rol', ['Student', 'Teacher', 'Administrative']);
        $table->foreignId('grupos_id')->nullable()->constrained('grupos')->cascadeOnDelete();
        $table->timestamps();
    });
}

public function down(): void
{
    Schema::dropIfExists('usuarios');
}
```

**Columns:**
- `id` - Primary key
- `nombre` - User's full name
- `email` - Unique email address
- `password` - Hashed password
- `rol` - Enum: 'Student', 'Teacher', or 'Administrative'
- `grupos_id` - Foreign key to grupos table (nullable, cascade delete)
- `created_at`, `updated_at` - Timestamps

```php
public function up(): void

```

**Columns:**
- `id` - Primary key

- `cursos_id` - Foreign key to cursos table (cascade delete)
- `created_at`, `updated_at` - Timestamps

## Migration Order

The migrations must run in this order to respect foreign key constraints:

1. System tables (cache, jobs)
2. `grupos` - No dependencies
3. `usuarios` - Depends on grupos
4. `cursos` - No dependencies
5. `course_groups` - Depends on both grupos and cursos

## Best Practices

<Note>
  Always backup your database before running migrations in production.
</Note>

<Warning>
  The `migrate:fresh` command will drop all tables. Use with caution!
</Warning>

<Tip>
  Use `migrate:status` to check which migrations have been run:
  
  ```bash
  php artisan migrate:status
  ```
</Tip>
