## Overview

The Robotics School application uses a relational database with four main tables representing users, groups, courses, and their relationships.

## Entity Relationship Diagram

The database follows this structure:

```
grupos (Groups)
  |
  |-- (One-to-Many) --> usuarios (Users)
  |
  |-- (Many-to-Many via course_groups) --> cursos (Courses)
```

## Core Tables

Stores student groupings by skill level (beginner, intermediate, advanced).

### usuarios (Users)

Stores all application users including students, teachers, and administrators.

<ResponseField name="id" type="bigint" required>
  Primary key, auto-increment
</ResponseField>

<ResponseField name="nombre" type="string">
  User's full name
</ResponseField>

<ResponseField name="email" type="string" required>
  User's email address (unique constraint)
</ResponseField>

<ResponseField name="password" type="string">
  Hashed password
</ResponseField>

<ResponseField name="rol" type="enum">

<ResponseField name="grupos_id" type="bigint">
  Foreign key to grupos table (nullable). Cascade delete enabled.

<ResponseField name="created_at" type="timestamp">

</ResponseField>

<ResponseField name="updated_at" type="timestamp">
  Record last update timestamp
</ResponseField>

## System Tables

Laravel includes additional system tables for caching and job queue management:

- **cache** / **cache_locks**: Application cache storage
- **jobs** / **job_batches** / **failed_jobs**: Queue and background job management

## Relationships

### One-to-Many

- A `grupo` has many `usuarios`
- A `usuario` belongs to one `grupo` (optional)

### Many-to-Many

- `grupos` and `cursos` are connected through the `course_groups` pivot table
- A group can have many courses
- A course can be assigned to many groups
