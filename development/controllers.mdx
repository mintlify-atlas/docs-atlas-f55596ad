## Overview

Controllers are responsible for handling HTTP requests and coordinating the application's response. The Robotics School application uses Laravel's resource controller pattern to organize CRUD operations.

## Resource Controllers

Resource controllers provide a standardized structure for handling common CRUD operations. Each resource controller includes seven standard methods:

## Implementing Controller Methods

Here's how to implement common controller patterns:

<Steps>
  <Step title="List All Resources">
    The `index()` method retrieves and displays all resources:

    ```php
    public function index()
    {
        $cursos = Curso::all();
        return view('cursos.index', compact('cursos'));
    }
    ```
  </Step>

  <Step title="Show Creation Form">
    The `create()` method displays the form:

    ```php
    public function create()
    {
        return view('cursos.create');
    }
    ```
  </Step>

  <Step title="Store New Resource">
    The `store()` method validates and saves data:

    ```php
    public function store(Request $request)
    {
        $validated = $request->validate([
            'nombre' => 'required|string|max:255',
            'descripcion' => 'required|string',
            'duracion' => 'required|integer|min:1',
        ]);

        $curso = Curso::create($validated);

        return redirect()->route('cursos.show', $curso)
            ->with('success', 'Curso creado exitosamente.');
    }
    ```
  </Step>

  <Step title="Display Single Resource">
    The `show()` method displays a specific resource:

    ```php
    public function show(string $id)
    {
        $curso = Curso::findOrFail($id);
        return view('cursos.show', compact('curso'));
    }
    ```
  </Step>

  <Step title="Show Edit Form">
    The `edit()` method displays the edit form:

    ```php
    public function edit(string $id)
    {
        $curso = Curso::findOrFail($id);
        return view('cursos.edit', compact('curso'));
    }
    ```
  </Step>

  <Step title="Update Resource">
    The `update()` method validates and updates data:

    ```php
    public function update(Request $request, string $id)
    {
        $curso = Curso::findOrFail($id);
        
        $validated = $request->validate([
            'nombre' => 'required|string|max:255',
            'descripcion' => 'required|string',
            'duracion' => 'required|integer|min:1',
        ]);

        $curso->update($validated);

        return redirect()->route('cursos.show', $curso)
            ->with('success', 'Curso actualizado exitosamente.');
    }
    ```
  </Step>

  <Step title="Delete Resource">
    The `destroy()` method removes the resource:

    ```php
    public function destroy(string $id)
    {
        $curso = Curso::findOrFail($id);
        $curso->delete();

        return redirect()->route('cursos.index')
            ->with('success', 'Curso eliminado exitosamente.');
    }
    ```
  </Step>
</Steps>

## Routing Resource Controllers

Register resource routes in `routes/web.php`:

<CodeGroup>
```php Single Resource
Route::resource('cursos', CursoController::class);
```

```php Multiple Resources
Route::resource('cursos', CursoController::class);
Route::resource('usuarios', UsuarioController::class);
```

```php Partial Resources
// Only certain methods
Route::resource('cursos', CursoController::class)
    ->only(['index', 'show']);

// Exclude certain methods
Route::resource('usuarios', UsuarioController::class)
    ->except(['destroy']);
```
</CodeGroup>

This single line generates all seven RESTful routes:

| HTTP Method | URI | Action | Route Name |
|------------|-----|--------|------------|
| GET | /cursos | index | cursos.index |
| GET | /cursos/create | create | cursos.create |
| POST | /cursos | store | cursos.store |
| GET | /cursos/{id} | show | cursos.show |
| GET | /cursos/{id}/edit | edit | cursos.edit |
| PUT/PATCH | /cursos/{id} | update | cursos.update |
| DELETE | /cursos/{id} | destroy | cursos.destroy |

## Request Validation

Validate incoming requests directly in controllers:

<CodeGroup>
```php Inline Validation
public function store(Request $request)
{
    $validated = $request->validate([
        'nombre' => 'required|string|max:255',
        'email' => 'required|email|unique:usuarios',
        'edad' => 'required|integer|min:8|max:18',
    ]);
    
    // Process validated data...
}
```

```php Custom Messages
public function store(Request $request)
{
    $validated = $request->validate([
        'nombre' => 'required|string|max:255',
        'email' => 'required|email|unique:usuarios',
    ], [
        'nombre.required' => 'El nombre es obligatorio.',
        'email.unique' => 'Este correo ya está registrado.',
    ]);
    
    // Process validated data...
}
```

```php Form Request Class
// Create: php artisan make:request StoreCursoRequest

public function store(StoreCursoRequest $request)
{
    // Data is automatically validated
    $validated = $request->validated();
    
    // Process validated data...
}
```
</CodeGroup>

## Dependency Injection

Inject dependencies into controller methods:

```php
use App\Models\Curso;
use App\Services\CursoService;
use Illuminate\Http\Request;

public function store(Request $request, CursoService $service)
{
    $validated = $request->validate([
        'nombre' => 'required|string|max:255',
    ]);
    
    $curso = $service->createCurso($validated);
    
    return redirect()->route('cursos.show', $curso);
}
```

## Response Types

Controllers can return various response types:

<CodeGroup>
```php View Response
public function show(string $id)
{
    $curso = Curso::findOrFail($id);
    return view('cursos.show', compact('curso'));
}
```

```php JSON Response
public function show(string $id)
{
    $curso = Curso::findOrFail($id);
    return response()->json($curso);
}
```

```php Redirect Response
public function store(Request $request)
{
    $curso = Curso::create($request->validated());
    
    return redirect()
        ->route('cursos.show', $curso)
        ->with('success', 'Curso creado exitosamente.');
}
```

```php Download Response
public function download(string $id)
{
    $curso = Curso::findOrFail($id);
    $pdf = $curso->generatePDF();
    
    return response()->download($pdf, 'curso.pdf');
}
```
</CodeGroup>

## Creating New Controllers

Generate controllers using Artisan:

<CodeGroup>
```bash Resource Controller
php artisan make:controller MatriculaController --resource
```

```bash API Resource Controller
php artisan make:controller Api/MatriculaController --api
```

```bash Controller with Model
php artisan make:controller MatriculaController --resource --model=Matricula
```
</CodeGroup>
